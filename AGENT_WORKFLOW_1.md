# 双代理协作工作流 | AGENT WORKFLOW

**文件版本**: v1.0  
**工作流模式**: Claude Code（核心） + Antigravity-Gemini（验证与推送）  
**适用项目**: 10kV配电网单相接地故障定位系统通讯故障修复

---

## 1. 代理职责边界

### Claude Code（核心执行代理）

**负责内容：**
- 读取和分析全部源代码
- 执行 `COMM_DEBUG_GUIDE.md` 中的所有排查提示词（P-01 ~ P-07）
- 填写 `BUG_TRACKING.md` 中的根因分析和修复方案
- 填写 `CODE_REVIEW_CHECKLIST.md` 中的实际结果列
- 输出具体的代码修改（修改前/修改后对比格式）
- 对修改方案作出嵌入式专业层面的合理性说明

**不负责内容：**
- 代码注释的中文规范化（交由Gemini）
- Git提交与推送操作（交由Gemini）
- 修改前后对比报告的最终排版（交由Gemini）
- 任何涉及可控硅触发逻辑的代码修改（需人工审批）

---

### Antigravity-Gemini（验证与交付代理）

**负责内容：**
- 接收 Claude Code 输出的修复方案
- 生成**修改前后对比报告**（格式见第4节）
- 为修改后的代码补全中文注释（符合项目注释规范）
- 执行 Git 提交（commit message 规范见第5节）
- 执行 Git 推送到目标分支
- 在 `BUG_TRACKING.md` 中填写验证记录和 commit hash

**不负责内容：**
- 独立判断代码逻辑正确性
- 在无 Claude Code 输出的情况下自行修改核心逻辑

## 3. Claude Code 启动指令

在 Antigravity 终端中，输入以下指令启动排查流程：

```
请阅读项目根目录下的以下文件，然后按顺序执行文件中规定的任务：

1. 首先阅读 PROJECT_OVERVIEW.md，理解系统架构和已知故障现象
2. 阅读 COMM_DEBUG_GUIDE.md，这是你的排查任务说明书
3. 阅读 CODE_REVIEW_CHECKLIST.md，这是你的审查清单
4. 阅读 BUG_TRACKING.md，了解需要排查的三个故障

然后从 COMM_DEBUG_GUIDE.md 中的 P-01 开始，逐步执行排查。
每完成一个步骤，等待我的确认再进行下一步。
所有输出使用中文。
注意：不得修改任何涉及可控硅触发控制的代码。
```

## 5. Git Commit Message 规范

```
格式：[类型] BUG-XXX: 简短描述（不超过50字）

类型：
  fix     - 修复bug
  refactor - 重构（不改变功能）
  docs    - 仅文档修改
  style   - 代码格式（不影响逻辑）

示例：
  fix BUG-001: 修正CAN过滤器ID配置，解决模块离线误报问题
  fix BUG-002: 增大CAN接收队列深度至32，解决多模块通讯丢帧
  fix BUG-003: 添加互斥锁保护模块在线状态数据结构
```

## 6. 人工审批节点

以下情况**必须邓总人工确认**后方可推送：

| 情况 | 原因 |
|------|------|
| 修改可控硅触发相关任何代码 | 直接影响10kV系统安全 |
| 修改CAN波特率 | 需同步更新全部设备固件 |
| 修改RT-Thread任务优先级 | 可能影响故障检测实时性 |
| 修改SPI通讯帧格式 | 需两块核心板同步更新 |
| 推送到主分支（main/master） | 主分支代码直接用于生产 |
